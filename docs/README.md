## ğŸ› ï¸ Development

### Local Development

1. Clone the repository
2. Navigate to the `docs` directory
3. Serve locally using any static server:

```bash
# Using Python
python -m http.server 8000

# Using Node.js serve
npx serve .

# Using PHP
php -S localhost:8000
```

4. Open `http://localhost:8000` in your browser

### Making Changes

1. **HTML**: Edit `index.html` for content changes
2. **Styling**: Modify `style.css` for design updates
3. **Functionality**: Update `script.js` for interactive features
4. **Installer**: Update `installer.sh` (automatically synced from root)

### CSS Architecture

The CSS uses modern features and follows these principles:
- **CSS Custom Properties**: For theming and consistency
- **Mobile-first**: Responsive design starting from mobile
- **Flexbox & Grid**: Modern layout techniques
- **Smooth animations**: CSS transitions and transforms
- **Dark mode ready**: Variables set up for future dark theme

### JavaScript Features

- **Vanilla JavaScript**: No frameworks or dependencies
- **Copy to clipboard**: Modern Clipboard API with fallback
- **Platform detection**: Automatic OS detection
- **Intersection Observer**: Smooth scroll animations
- **Error handling**: Graceful degradation for older browsers

## ğŸš€ Deployment

The site is automatically deployed to GitHub Pages using GitHub Actions when changes are pushed to the main branch.

### Deployment Process

1. **Build**: Minifies CSS, JavaScript, and HTML
2. **Test**: Validates HTML and checks installer script
3. **Security**: Scans for potential security issues
4. **Deploy**: Uploads to GitHub Pages

### Manual Deployment

If you need to deploy manually:

1. Ensure all files are in the `docs` directory
2. Enable GitHub Pages in repository settings
3. Set source to "Deploy from a branch" and select `main` branch, `/docs` folder
